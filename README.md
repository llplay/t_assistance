# AI 智能教案生成系统

这是一个基于大语言模型（GLM-4）的高职教案智能生成平台。教师只需输入课程基本信息和教学流程节点，系统即可自动生成详细的教案内容，并支持导出为标准的 Word 文档。

---

## 🚀 主要功能

-   **智能内容生成**：利用 AI 模型（智谱 GLM-4）根据课程、年级、教材及教学流程，自动补全教学目标、学情分析、教学重难点等详细内容。
-   **灵活教学流程**：支持用户自定义教学流程节点（如：明确任务、分析任务、评价总结等），AI 将围绕这些节点生成对应的教师与学生活动。
-   **Word 文档导出**：一键将生成的教案填充到预设的专业 Word 模板中，保持格式统一。
-   **流程图支持**：支持上传教学流程图片，并将其自动嵌入到生成的 Word 文档中。
-   **多模式编辑**：前端提供友好的 UI 表单界面，同时也支持 JSON 编辑模式，方便高级用户直接调整原始数据。
-   **用户权限管理**：包含用户注册、登录及密码修改功能，确保教案数据的私密性。

## 🏗️ 系统架构

系统采用典型的**前后端分离**架构：

### 后端 (Backend)
-   **框架**：Flask (Python)
-   **数据库**：SQLite (通过 SQLAlchemy ORM 管理)
-   **AI 集成**：智谱 AI SDK (GLM-4-Flash 模型)
-   **文档处理**：python-docx (用于解析和生成 Word 文档)
-   **认证**：JWT (JSON Web Token)

### 前端 (Frontend)
-   **框架**：React + TypeScript
-   **UI 组件库**：Ant Design (antd)
-   **图标库**：Lucide React / Ant Design Icons
-   **Markdown 渲染**：react-markdown

## 🔄 数据流转

1.  **输入阶段**：教师在前端表单输入课程信息（专业、年级、内容、学时等）并设定教学流程节点。
2.  **生成阶段**：
    -   前端将数据发送至后端 `/api/generate` 接口。
    -   后端组装详细的 Prompt，调用智谱 AI API。
    -   AI 以 **流式（Streaming）** 方式返回生成的教案 JSON 数据。
    -   前端实时刷新并展示 Markdown 格式的教案预览。
3.  **下载/转换阶段**：
    -   用户点击下载，前端将教案内容及图片（FormData）发送至 `/api/download`。
    -   后端解析教案 JSON，并加载本地的 `教案模板.docx`。
    -   系统遍历 Word 模板中的占位符（如 `##教学内容##`），使用 AI 生成的内容进行精确替换。
    -   如果是流程图位置，则将用户上传（或默认）的图片插入单元格。
    -   最终生成 `.docx` 文件流返回给浏览器。

## 📝 模板占位符说明

### 教案模板（`教案模板.docx`）
使用 `##键名##` 形式的占位符，后端自动将 AI 生成的 JSON 对应字段填入其中。
例如：`##教学目标##知识目标##` 会对应填充 JSON 中 `教学目标` 下的 `知识目标` 内容。

## 📋 授课计划功能（`schedule_template.docx`）

### 模板生成原理

**脚本**：`make_schedule_template.py`

将原始授课计划 `.docx` 中的所有可变内容替换为 `{{占位符}}`，格式和样式完全保留不变。后端填充时，只做文字替换，不改动任何排版。

### 文档结构

| 部分 | 内容 | 处理方式 |
|---|---|---|
| 封面段落 | 学年学期、系部、教研室等信息行 | 替换冒号后的内容为占位符 |
| 编写说明页 | 六条固定说明文字 | **不修改**，内容固定 |
| 课程信息表（表格0，8行×4列） | 各类学时、教材信息、课程简介 | 替换值单元格为占位符 |
| 排课表（表格1，40行×10列） | 每节课的月份/周次/内容/重难点等 | 保留2行表头+1行模板行，删除其余38行数据 |
| 签名行 | 教研室主任/系部主任等签名处 | **不修改** |

### 排课表工作原理

原始文档中排课表有 40 行（2行表头 + 38行数据）。模板化后只保留：

```
行0: 月份 | 周次 | 星期 | 节次 | 教学内容 | 教学学时 |  形式  | 教学重点难点 | 课外作业
行1: (同上，子表头)                                    | 理论|实践 |
行2: {{月份}} | {{周次}} | {{星期}} | {{节次}} | {{教学内容}} | {{教学学时}} | {{理论}} | {{实践}} | {{重难点}} | {{课外作业}}
```

**后端填充逻辑**：读取模板行2的格式，按 AI 生成的 N 条记录，复制 N 次该行并逐行填入内容，完整保留原始字体、对齐、边框。

> ⚠️ `python-docx` 读合并单元格时会重复返回同一个 cell 对象，脚本通过 `id(cell._tc)` 去重，确保每列只写一次占位符。

### 占位符列表

**封面段落：**
| 占位符 | 含义 |
|---|---|
| `{{学年学期}}` | 如：2024-2025学年第一学期 |
| `{{系部}}` | 如：信息工程系 |
| `{{教研室}}` | 如：人工智能教研室 |
| `{{课程名称}}` | 课程名 |
| `{{教学层次}}` | 如：大专 |
| `{{授课班级}}` | 如：D23人工智能1班，2班 |
| `{{制订人}}` | 制订教师姓名 |
| `{{其他任课教师}}` | 其他教师姓名 |
| `{{制订日期}}` | 如：2024年8月22日 |

**课程信息表：**
| 占位符 | 含义 |
|---|---|
| `{{课程总学时}}` / `{{机动学时}}` | 学时数字 |
| `{{理论教学学时}}` / `{{实践教学学时}}` | 学时数字 |
| `{{周学时数}}` / `{{教学周数}}` | 数字 |
| `{{复习学时}}` / `{{考核学时}}` | 数字 |
| `{{教学大纲}}` | 大纲名称及版本 |
| `{{基本教材}}` | 教材名称、作者、出版社、年份 |
| `{{参考书}}` | 参考书信息 |
| `{{课程简介}}` | 课程描述大段文字 |

**排课表数据行：**
| 占位符 | 含义 |
|---|---|
| `{{月份}}` `{{周次}}` `{{星期}}` `{{节次}}` | 时间信息 |
| `{{教学内容}}` | 本节课教学内容（可含换行） |
| `{{教学学时}}` `{{理论}}` `{{实践}}` | 课时分配 |
| `{{重难点}}` | 教学重点和难点 |
| `{{课外作业}}` | 课后作业（可为空） |
